# TypeScript Config Package

The TypeScript config package provides shared TypeScript configuration and type definitions for the Waylio healthcare platform. It includes healthcare-specific types, strict type checking rules, and development tooling configurations optimized for the monorepo structure.

## Installation

```bash
# Install the TypeScript config package
pnpm add @repo/typescript-config

# Peer dependencies (automatically installed in monorepo)
pnpm add typescript @types/node @types/react @types/react-dom
```

## Overview

The TypeScript config package ensures consistent TypeScript configuration across all applications and packages in the Waylio monorepo. It provides healthcare-specific type definitions, strict type checking rules, and optimized compiler settings for both development and production builds.

  // Utility types for better type safety
  export type Prettify<T> = {
    [K in keyof T]: T[K];
  } & {};
  
  export type DeepPartial<T> = {
    [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
  };
  
  export type DeepRequired<T> = {
    [P in keyof T]-?: T[P] extends object ? DeepRequired<T[P]> : T[P];
  };
  
  export type NonEmptyArray<T> = [T, ...T[]];
  
  export type ValueOf<T> = T[keyof T];
  
  export type KeysOfType<T, U> = {
    [K in keyof T]: T[K] extends U ? K : never;
  }[keyof T];
  
  export type RequiredKeys<T> = {
    [K in keyof T]-?: {} extends Pick<T, K> ? never : K;
  }[keyof T];
  
  export type OptionalKeys<T> = {
    [K in keyof T]-?: {} extends Pick<T, K> ? K : never;
  }[keyof T];
  
  // Date utility types

## Best Practices

### TypeScript Guidelines

<AccordionGroup>
<Accordion title="Type Safety">
  - Use strict mode and enable all strict type checking options
  - Prefer type assertions over any types
  - Use branded types for IDs to prevent mixing different entity IDs
  - Implement proper error handling with typed error objects
  - Use discriminated unions for complex state management
</Accordion>

<Accordion title="Code Organization">
  - Organize types by domain (healthcare, api, utils)
  - Use barrel exports for clean imports
  - Implement consistent naming conventions
  - Document complex types with JSDoc comments
  - Use utility types to reduce code duplication
</Accordion>

<Accordion title="Performance">
  - Enable incremental compilation for faster builds
  - Use composite projects for better build performance
  - Implement proper tree shaking with ES modules
  - Optimize path mapping for faster module resolution
  - Use type-only imports where appropriate
</Accordion>
</AccordionGroup>

<Note>
TypeScript configuration in healthcare applications should prioritize type safety and maintainability. Strict type checking helps prevent runtime errors that could impact patient care.
</Note>
