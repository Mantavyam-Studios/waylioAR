# Design System Package

The design system package provides a comprehensive UI component library that ensures visual consistency and accessibility across all Waylio applications. Built with React, TypeScript, and Tailwind CSS, it includes both foundational components and healthcare-specific patterns.

## Installation

```bash
# Install the design system package
pnpm add @repo/design-system

# Peer dependencies (automatically installed in monorepo)
pnpm add react react-dom tailwindcss
```

## Overview

The design system serves as the single source of truth for UI components, design tokens, and interaction patterns across the Waylio platform. It's designed with accessibility, healthcare workflows, and developer experience in mind.

<Accordion title="AppointmentCalendar">
  **Purpose**: Calendar view for appointment scheduling
  
  ```typescript
  import { Calendar } from '@repo/design-system';
  import { useState } from 'react';
  
  interface Appointment {
    id: string;
    patientName: string;
    time: string;
    duration: number;
    type: string;
    status: 'scheduled' | 'confirmed' | 'in-progress' | 'completed';
  }
  
  interface AppointmentCalendarProps {
    appointments: Appointment[];
    onDateSelect?: (date: Date) => void;
    onAppointmentClick?: (appointment: Appointment) => void;
  }
  
  export function AppointmentCalendar({ 
    appointments, 
    onDateSelect,
    onAppointmentClick 
  }: AppointmentCalendarProps) {
    const [selectedDate, setSelectedDate] = useState<Date>(new Date());
    
    const handleDateSelect = (date: Date | undefined) => {
      if (date) {
        setSelectedDate(date);
        onDateSelect?.(date);
      }
    };
    
    const dayAppointments = appointments.filter(apt => 
      isSameDay(new Date(apt.date), selectedDate)
    );
    
    return (
      <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
        <div>
          <Calendar
            mode="single"
            selected={selectedDate}
            onSelect={handleDateSelect}
            className="rounded-md border"
            modifiers={{
              hasAppointments: (date) => 
                appointments.some(apt => isSameDay(new Date(apt.date), date))
            }}
            modifiersStyles={{
              hasAppointments: { 
                backgroundColor: 'rgb(59 130 246 / 0.1)',
                color: 'rgb(59 130 246)'
              }
            }}
          />
        </div>
        
        <div className="space-y-4">
          <h3 className="font-semibold">
            Appointments for {format(selectedDate, 'MMMM d, yyyy')}
          </h3>
          
          {dayAppointments.length === 0 ? (
            <p className="text-muted-foreground">No appointments scheduled</p>
          ) : (
            <div className="space-y-2">
              {dayAppointments.map((appointment) => (
                <Card 
                  key={appointment.id}
                  className="cursor-pointer hover:shadow-sm transition-shadow"
                  onClick={() => onAppointmentClick?.(appointment)}
                >
                  <CardContent className="p-4">
                    <div className="flex items-center justify-between">
                      <div>
                        <p className="font-medium">{appointment.patientName}</p>
                        <p className="text-sm text-muted-foreground">
                          {appointment.time} • {appointment.duration}min • {appointment.type}
                        </p>
                      </div>
                      <Badge variant={
                        appointment.status === 'confirmed' ? 'default' :
                        appointment.status === 'in-progress' ? 'secondary' :
                        appointment.status === 'completed' ? 'outline' : 'secondary'
                      }>
                        {appointment.status}
                      </Badge>
                    </div>
                  </CardContent>
                </Card>
              ))}
            </div>
          )}
        </div>
      </div>
    );
  }
  ```

## Best Practices

### Accessibility Guidelines

<AccordionGroup>
<Accordion title="Keyboard Navigation">
  - All interactive elements must be keyboard accessible
  - Implement proper focus management
  - Use semantic HTML elements
  - Provide skip links for navigation
  - Test with screen readers regularly
</Accordion>

<Accordion title="Color and Contrast">
  - Maintain WCAG AA contrast ratios (4.5:1 for normal text)
  - Don't rely solely on color to convey information
  - Provide alternative text for images
  - Use consistent color patterns
  - Test with color blindness simulators
</Accordion>

<Accordion title="Healthcare-Specific Considerations">
  - Use clear, medical-appropriate language
  - Implement proper error handling for critical actions
  - Provide confirmation dialogs for destructive actions
  - Ensure data privacy in UI components
  - Follow healthcare industry design patterns
</Accordion>
</AccordionGroup>

<Note>
The design system is the foundation of user experience across Waylio. Always prioritize accessibility, consistency, and healthcare-specific needs when using or extending components.
</Note>
