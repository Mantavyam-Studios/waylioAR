# API Reference

Welcome to the Waylio API reference documentation. This comprehensive guide covers all available endpoints, authentication methods, and integration patterns for building healthcare applications with the Waylio platform.

## Base URL

All API requests should be made to:

```
https://api.waylio.com/v1
```

For development and testing:

```
http://localhost:3000/api/v1
```

## Authentication

### Bearer Token Authentication

The Waylio API uses Bearer token authentication. Include your API token in the Authorization header:

```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
     https://api.waylio.com/v1/patients
```

### Getting Your API Token

1. Log in to your Waylio dashboard
2. Navigate to Settings > API Keys
3. Generate a new API key for your application
4. Store the key securely and never expose it in client-side code

### API Features

- **RESTful Design**: Standard HTTP methods and status codes
- **JSON Responses**: All responses are in JSON format
- **Rate Limiting**: 1000 requests per hour per API key
- **Pagination**: Cursor-based pagination for large datasets
- **Filtering**: Advanced filtering and search capabilities
- **Webhooks**: Real-time event notifications
- **HIPAA Compliant**: End-to-end encryption and audit logging

## Quick Start

### 1. Authentication Setup

```javascript
// JavaScript/Node.js
const WAYLIO_API_TOKEN = 'your_api_token_here';
const BASE_URL = 'https://api.waylio.com/v1';

const headers = {
  'Authorization': `Bearer ${WAYLIO_API_TOKEN}`,
  'Content-Type': 'application/json',
};
```

### 2. Make Your First Request

```javascript
// Fetch all patients
async function getPatients() {
  const response = await fetch(`${BASE_URL}/patients`, {
    method: 'GET',
    headers: headers,
  });
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  const data = await response.json();
  return data;
}

// Usage
getPatients()
  .then(patients => console.log(patients))
  .catch(error => console.error('Error:', error));
```

### 3. Create a New Patient

```javascript
// Create a new patient
async function createPatient(patientData) {
  const response = await fetch(`${BASE_URL}/patients`, {
    method: 'POST',
    headers: headers,
    body: JSON.stringify(patientData),
  });
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  const patient = await response.json();
  return patient;
}

// Example patient data
const newPatient = {
  firstName: 'John',
  lastName: 'Doe',
  dateOfBirth: '1990-01-15',
  gender: 'male',
  email: 'john.doe@example.com',
  phoneNumber: '+1-555-123-4567',
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA',
    zipCode: '12345',
    country: 'US'
  }
};

createPatient(newPatient)
  .then(patient => console.log('Created patient:', patient))
  .catch(error => console.error('Error:', error));
```

## HTTP Status Codes

| Status Code | Description |
|-------------|-------------|
| `200` | OK - Request successful |
| `201` | Created - Resource created successfully |
| `400` | Bad Request - Invalid request parameters |
| `401` | Unauthorized - Invalid or missing API token |
| `403` | Forbidden - Insufficient permissions |
| `404` | Not Found - Resource not found |
| `409` | Conflict - Resource already exists |
| `422` | Unprocessable Entity - Validation errors |
| `429` | Too Many Requests - Rate limit exceeded |
| `500` | Internal Server Error - Server error |
| `503` | Service Unavailable - Service temporarily unavailable |

## Rate Limiting

The Waylio API implements rate limiting to ensure fair usage:

- **Standard Plan**: 1,000 requests per hour
- **Professional Plan**: 10,000 requests per hour
- **Enterprise Plan**: 100,000 requests per hour

Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1642694400
```

## Pagination

Large datasets are paginated using cursor-based pagination:

```bash
# First page
GET /api/v1/patients?limit=20

# Next page using cursor
GET /api/v1/patients?limit=20&cursor=cursor_abc123

# Previous page using cursor
GET /api/v1/patients?limit=20&cursor=cursor_xyz789&direction=prev
```

## Filtering and Search

### Query Parameters

```bash
# Filter patients by gender
GET /api/v1/patients?gender=female

# Search patients by name
GET /api/v1/patients?search=john

# Filter by date range
GET /api/v1/appointments?startDate=2024-01-01&endDate=2024-01-31

# Multiple filters
GET /api/v1/patients?gender=male&ageMin=18&ageMax=65
```

### Advanced Search

```bash
# Full-text search across multiple fields
GET /api/v1/patients?q=john+doe+cardiology

# Filter by nested fields
GET /api/v1/patients?address.state=CA&address.city=Los+Angeles
```

## Webhooks

Subscribe to real-time events using webhooks:

### Webhook Events

- `patient.created` - New patient registered
- `patient.updated` - Patient information updated
- `appointment.created` - New appointment scheduled
- `appointment.cancelled` - Appointment cancelled
- `prescription.created` - New prescription issued
- `lab_result.available` - Lab results ready

### Webhook Configuration

```javascript
// Configure webhook endpoint
const webhookConfig = {
  url: 'https://your-app.com/webhooks/waylio',
  events: ['patient.created', 'appointment.created'],
  secret: 'your_webhook_secret'
};

// Register webhook
fetch(`${BASE_URL}/webhooks`, {
  method: 'POST',
  headers: headers,
  body: JSON.stringify(webhookConfig)
});
```

## Error Handling

### Common Error Codes

| Error Code | Description | Resolution |
|------------|-------------|------------|
| `INVALID_API_KEY` | API key is invalid or expired | Check your API key in dashboard |
| `INSUFFICIENT_PERMISSIONS` | Missing required permissions | Contact support for access |
| `VALIDATION_ERROR` | Request validation failed | Check request parameters |
| `RESOURCE_NOT_FOUND` | Requested resource doesn't exist | Verify resource ID |
| `RATE_LIMIT_EXCEEDED` | Too many requests | Wait before retrying |
| `HIPAA_VIOLATION` | Request violates HIPAA compliance | Review data access policies |

### Error Handling Best Practices

```javascript
async function handleApiRequest() {
  try {
    const response = await fetch(`${BASE_URL}/patients`, {
      method: 'GET',
      headers: headers,
    });
    
    if (!response.ok) {
      const error = await response.json();
      
      switch (error.error.code) {
        case 'RATE_LIMIT_EXCEEDED':
          // Wait and retry
          await new Promise(resolve => setTimeout(resolve, 60000));
          return handleApiRequest();
          
        case 'INVALID_API_KEY':
          // Refresh API key
          throw new Error('API key needs to be refreshed');
          
        default:
          throw new Error(error.error.message);
      }
    }
    
    return await response.json();
  } catch (error) {
    console.error('API request failed:', error);
    throw error;
  }
}
```
